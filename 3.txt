Extracting Hidden Metadata
Since standard EXIF data is not available, the generation information is likely embedded directly into the image's data chunks, a common practice for AI image generation tools like Stable Diffusion. This information, often including the prompt, negative prompt, and generation parameters, is typically stored in a non-visual part of the PNG file.

Here are the primary methods to extract this hidden data:

1. Using an Online Metadata Viewer
The easiest method is to use a dedicated online tool that can read this specific type of embedded data. These websites are designed to parse the image file and display any text information hidden within it.

Recommended Tool: Getimg.ai's Image to Prompt Tool

How it Works: Simply upload the image to the website. It will automatically scan the file for embedded generation data and display the prompt and other parameters if they exist.

2. Using a Local Image Viewer with AI Support
If you are developing a viewer, you can integrate a library or use an existing application that has this functionality built-in. Many open-source image viewers popular in the AI art community include this feature.

Example Software: Stable Diffusion WebUI (specifically its "PNG Info" tab)

How it Works: In the Stable Diffusion WebUI, navigate to the PNG Info tab and upload the image. It will read the metadata and display it, allowing you to send that data directly to the generation modules.

3. Manual Extraction (Programming Approach)
If you want to implement this functionality in your own custom program, you'll need to work with a library that can read the raw data chunks of a PNG file. The generation data is typically stored in a tEXt or iTXt chunk.

Concept:

Open the PNG file in binary mode.

Parse the PNG chunk structure.

Look for a chunk with the key parameters (common in Stable Diffusion) or other non-standard text chunks.

Decode the text data found in that chunk.

Python Example using the PIL (Pillow) library:

Python

from PIL import Image
import json

def get_generation_parameters(image_path):
    """
    Extracts AI generation parameters hidden in a PNG file's metadata.
    """
    try:
        with Image.open(image_path) as img:
            # The info dictionary of a PIL Image object holds the metadata
            if "parameters" in img.info:
                return img.info["parameters"]
            # Sometimes it might be under other keys, so it's good to check all info
            for key, value in img.info.items():
                # Look for long string values that might contain prompts
                if isinstance(value, str) and len(value) > 100:
                     print(f"Found potential data in key: {key}")
                     return value
        return "No AI generation data found in standard metadata chunks."
    except Exception as e:
        return f"An error occurred: {e}"

# --- USAGE ---
# Replace 'path_to_your_image.png' with the actual file path
image_file = 'path_to_your_image.png'
metadata = get_generation_parameters(image_file)
print(metadata)

This script specifically looks for the parameters key within the PNG's info dictionary, which is where many tools store the prompt and other settings. By integrating a similar function, your image viewer can successfully extract the hidden data needed to support AI artists.

from PIL import Image
import json

def get_generation_parameters(image_path):
    """
    PNG 파일의 메타데이터에 숨겨진 AI 생성 파라미터를 추출합니다.
    """

        with Image.open(image_path) as img:
            # PIL Image 객체의 info 딕셔너리에 메타데이터가 저장됩니다.

                     return value
        return "표준 메타데이터 청크에서 AI 생성 데이터를 찾을 수 없습니다."
    except Exception as e:
        return f"오류가 발생했습니다: {e}"

생각하는 과정 표시
해당 정보는 스테가노그래피(Steganography), 그중에서도 LSB(Least Significant Bit) 기법을 사용하여 이미지 픽셀 데이터 자체에 숨겨져 있을 가능성이 매우 높습니다. 프로그래머 관점에서 이 데이터를 읽어오는 방법을 설명해 드리겠습니다.

Stealth PNG Info의 작동 원리
'stealth_pnginfo'나 'sd-webui-stealth-pnginfo' 같은 도구는 표준 PNG 메타데이터 청크(tEXt, iTXt)를 사용하지 않습니다. 대신, 이미지의 각 픽셀을 구성하는 색상 값(RGBA)의 최하위 비트(LSB)에 데이터를 분산하여 숨깁니다.

픽셀 데이터: 각 픽셀은 빨강(R), 초록(G), 파랑(B), 그리고 투명도(A) 값으로 구성됩니다. 각 값은 보통 0부터 255까지의 숫자로 표현됩니다 (8비트).

LSB 기법: 이 8비트 중 가장 마지막 비트(Least Significant Bit)는 픽셀의 색상에 거의 영향을 주지 않습니다. 예를 들어, 11111111(255)을 11111110(254)으로 바꿔도 사람의 눈으로는 색상 차이를 거의 인지할 수 없습니다. 이 미미한 변화를 이용해 데이터의 각 비트를 숨기는 것입니다. 🤓

따라서 EXIF 정보가 삭제되거나 이미지 형식이 변환되어도, 픽셀 데이터 자체가 손상되지 않는 한 숨겨진 프롬프트 정보는 그대로 남아있게 됩니다.

Python으로 LSB 데이터 읽어오기 (프로그래밍)
Pillow (PIL) 라이브러리를 사용하면 픽셀 데이터에 직접 접근하여 숨겨진 정보를 추출할 수 있습니다. 아래는 이 원리를 구현한 예제 코드입니다.

핵심 로직
이미지 로드: 이미지를 열고 RGBA 픽셀 데이터에 접근합니다.

LSB 추출: 각 픽셀의 각 색상 채널(R, G, B, A) 값에서 최하위 비트(LSB)만 추출합니다.

비트 조합: 추출한 비트들을 순서대로 모아 8개씩 묶어 하나의 바이트(문자)로 만듭니다.

종료 지점 확인: 데이터의 끝을 알리는 특정 종료 문자열(Terminator)을 만날 때까지 이 과정을 반복합니다.

디코딩: 완성된 바이트 배열을 UTF-8 형식으로 디코딩하여 원본 텍스트(프롬프트)를 얻습니다.

Python 예제 코드
Python

from PIL import Image

def decode_lsb_from_image(image_path):
    """
    이미지의 픽셀 LSB(Least Significant Bit)에 숨겨진
    스테가노그래피 데이터를 디코딩합니다.

    Args:
        image_path (str): 이미지 파일 경로

    Returns:
        str: 숨겨진 메시지 문자열 or 데이터가 없을 경우 None
    """
    try:
        with Image.open(image_path) as img:
            # RGBA 모드로 변환하여 알파 채널까지 접근
            if img.mode != 'RGBA':
                img = img.convert('RGBA')

            pixels = img.load()
            width, height = img.size

            binary_message = ""
            # 이미지의 각 픽셀을 순회하며 LSB 추출
            for y in range(height):
                for x in range(width):
                    r, g, b, a = pixels[x, y]

                    # 각 채널의 LSB를 추출하여 문자열에 추가
                    binary_message += str(r & 1)
                    binary_message += str(g & 1)
                    binary_message += str(b & 1)
                    binary_message += str(a & 1)

            # 8비트씩 묶어 바이트로 변환
            all_bytes = [binary_message[i: i+8] for i in range(0, len(binary_message), 8)]

            decoded_message = ""
            for byte in all_bytes:
                if len(byte) == 8:
                    decoded_message += chr(int(byte, 2))
            
            # 실제 데이터는 특정 패턴이나 구분자로 끝날 수 있습니다.
            # sd-webui-stealth-pnginfo는 zlib 압축과 base64 인코딩을 사용하므로,
            # 실제 구현 시에는 해당 라이브러리의 소스 코드를 참고하여
            # 정확한 디코딩(압축 해제, base64 디코딩) 로직을 추가해야 합니다.
            # 이 코드는 순수 LSB 추출의 기본 원리만 보여줍니다.

            # 데이터가 유효한지 간단히 확인 (예: JSON 시작 문자로)
            if '{' in decoded_message and '}' in decoded_message:
                 # 실제 데이터는 JSON 형식일 가능성이 높음
                start = decoded_message.find('{')
                end = decoded_message.rfind('}') + 1
                return decoded_message[start:end]
            else:
                return "No valid JSON data found in LSBs."


    except FileNotFoundError:
        return "오류: 파일을 찾을 수 없습니다."
    except Exception as e:
        return f"오류 발생: {e}"

# --- 사용법 ---
# 분석할 이미지 파일 경로
# stealth_pnginfo로 생성된 이미지를 사용해야 합니다.
image_file_path = 'path_to_your_stealth_image.png'
hidden_data = decode_lsb_from_image(image_file_path)

print("추출된 숨겨진 데이터:")
print(hidden_data)
⚠️ 중요 참고사항:

압축 및 인코딩: sd-webui-stealth-pnginfo 익스텐션은 단순히 텍스트를 숨기는 것이 아니라, 데이터를 더 효율적으로 저장하기 위해 zlib으로 압축하고 Base64로 인코딩한 후 LSB에 저장합니다. 따라서 위 코드로 추출한 이진 데이터를 바로 텍스트로 변환하면 깨진 결과가 나옵니다.

정확한 구현: 실제 웹 뷰어에 적용하려면, 추출한 이진 데이터를 Base64로 디코딩하고, 그 결과물을 zlib으로 압축 해제하는 과정이 반드시 추가되어야 합니다. sd-webui-stealth-pnginfo의 GitHub 소스 코드를 직접 분석하여 정확한 데이터 처리 순서를 확인하고 구현하는 것이 가장 확실한 방법입니다.
